# PRD: 국회의원 의정활동 투명성 플랫폼 (Vote the Boat)

## 1. 개요

### 1.1 프로젝트 요약
"내가 뽑은 의원은 무엇을 하고 있는가?"라는 질문에 답하는 시민 감시 웹 플랫폼.
지도 기반으로 내 지역구 국회의원·지방자치단체장의 의정활동(법안 발의, 표결 참여율, 재산, 전과 등)을 시각화한다.

### 1.2 핵심 차별점
| 구분 | 기존 서비스 (kassembly 등) | Vote the Boat |
|---|---|---|
| 진입 흐름 | 전체 의원 랭킹 → 검색 | **내 지역구 → 내 의원** |
| 핵심 가치 | 의원 간 비교·순위 | 내 대리인의 활동 감시 |
| 사용 시점 | 관심 있을 때 수시 방문 | 선거 전·법안 논란 시 자연 유입 |

### 1.3 목표
- 유권자가 자신의 지역구 의원 활동 정보를 **3클릭 이내**에 확인
- 선거 시즌 트래픽 급증에 대응 가능한 구조 설계
- 공공 데이터 기반으로 정확하고 최신 상태의 정보 제공

## 2. 타겟 사용자

### 2.1 1차 타겟: 정치 관심 시민 (20~40대)
- 투표는 하지만 내가 뽑은 의원의 활동을 모르는 유권자
- 뉴스에서 법안 논란을 접하고 "우리 의원은 어떻게 투표했지?" 궁금한 사람
- 선거철에 후보 비교 정보를 찾는 유권자

### 2.2 2차 타겟: 지역 이슈 당사자
- 재개발, 교통, 환경 등 지역 현안에 직접 영향받는 주민
- 지역구 의원에게 민원·청원을 넣으려는 사람
- 시민단체·언론 활동가

### 2.3 3차 타겟: 선거 시즌 유입자
- 선거 직전 후보 전과·재산·공약 이행률을 한눈에 비교하고 싶은 유권자

## 4. 기능 요구사항

### 4.1 지도 기반 선거구 탐색 (P0)
- Leaflet 기반 대한민국 지도에 선거구 경계(GeoJSON) 오버레이
- 선거구 클릭 시 해당 지역 국회의원 정보 패널 표시
- 선거구 GeoJSON 데이터: `OhmyNews/2024_22_elec_map` (SGG_Code 기준 매핑)
- 지도 줌 레벨에 따라 시도 → 시군구 → 선거구 단계적 표시

### 4.2 의원 프로필 (P0)
- 기본 정보: 이름, 정당, 선거구, 당선 횟수, 위원회, 사진
- 고유 식별자: 열린국회정보 `MONA_CD` 기준 통일
- 연락처: 사무실 전화, 이메일, 홈페이지

### 4.3 법안 발의 내역 (P0)
- 의원별 발의 법안 목록 (최신순, 페이지네이션)
- 법안명, 발의일, 처리 상태(계류/가결/부결/폐기), 공동발의 여부
- 발의 건수 통계 (대수별, 연도별)

### 4.4 표결 참여율 (P0)
- 본회의 표결 참여율 수치 표시 (출석률 API 미제공 → 표결 참여율로 대체)
- 참여율 추이 차트 (Recharts)
- 표결별 상세 내역: 찬성/반대/기권/불참

### 4.5 의원 검색 (P0)
- 이름, 정당, 지역 기반 통합 검색
- 자동완성 지원
- 검색 결과에서 의원 상세 페이지 직접 이동

### 4.6 재산 신고 내역 (P1)
- 선거 기간 공개 데이터 → 선거 직후 배치 수집하여 DB 영구 저장
- 항목별 재산 내역 표시 (부동산, 예금, 유가증권 등)

### 4.7 전과 기록 (P1)
- 선거 기간 공개 데이터 → 선거 직후 배치 수집하여 DB 영구 저장
- 죄명, 형량, 확정일 표시

### 4.8 주소 기반 선거구 찾기 (P1)
- Kakao 주소 검색 API로 사용자 주소 입력 → 좌표 변환
- 좌표 → 선거구 매핑 → 해당 의원 정보 표시

## 5. 비기능 요구사항

### 5.1 성능
- 지도 초기 로딩: 3초 이내 (GeoJSON 최적화, 타일 캐싱)
- API 응답 시간: 평균 200ms 이내
- 의원 프로필, 선거구-의원 매핑은 Redis 캐시 적용

### 5.2 데이터 최신성
- 의원정보·법안·표결: 매일 새벽 3시 Spring Batch 자동 수집
- 수집 순서: 의원정보 → 법안 → 표결 (의존성 순)
- 재시도: 실패 시 3회 재시도 (`faultTolerant().retry(Exception.class).retryLimit(3)`)
- 전과·재산: 선거 직후 수동 트리거 1회성 배치

### 5.3 확장성
- 선거 시즌 트래픽 급증(평소 대비 10배 이상) 대응
- 정적 데이터 CDN 캐싱, API 레이어 수평 확장 가능 구조

### 5.4 SEO
- 선거구별 랜딩 페이지 생성 (SSR/SSG)
- 핵심 키워드: `{지역명} 국회의원`, `국회의원 출석률`, `국회의원 전과 조회`
- Open Graph 메타태그 (의원명, 선거구, 표결 참여율 포함)

### 5.5 접근성
- 모바일 반응형 (지도 터치 인터랙션 지원)
- 시각 장애인을 위한 스크린 리더 대응 (alt 텍스트, ARIA 라벨)

## 6. 기술 아키텍처

### 6.1 기술 스택
| 영역 | 기술 |
|---|---|
| 백엔드 | Java 17, Spring Boot 3.x, Spring Batch, Spring Data JPA, QueryDSL |
| 데이터베이스 | PostgreSQL (주 DB), Redis (캐시) |
| 프론트엔드 | Next.js 14, Tailwind CSS, React-Leaflet, Recharts |
| 배포 | Vercel (프론트엔드), Railway (백엔드 + DB) |

### 6.2 시스템 구성도
```
┌─────────────┐     ┌─────────────────┐     ┌──────────────┐
│  Next.js     │────▶│  Spring Boot    │────▶│  PostgreSQL  │
│  (Vercel)    │     │  REST API       │     │  (Railway)   │
└─────────────┘     │  (Railway)      │     └──────────────┘
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐     ┌──────────────┐
                    │  Spring Batch   │     │    Redis      │
                    │  Collector      │     │   (Cache)     │
                    └────────┬────────┘     └──────────────┘
                             │
                    ┌────────▼────────┐
                    │  공공 API        │
                    │  (data.go.kr)   │
                    └─────────────────┘
```

### 6.3 데이터 수집 파이프라인
```
[매일 03:00 Cron]
  ├─ Step 1: 의원정보 수집 (열린국회정보 API → member 테이블)
  ├─ Step 2: 법안 수집 (의안정보 API → bill 테이블)
  └─ Step 3: 표결 수집 (표결정보 API → vote 테이블)

[수동 트리거 - 선거 직후]
  ├─ Job: 재산 수집 (선관위 API → asset 테이블)
  └─ Job: 전과 수집 (선관위 API → criminal 테이블)
```

### 6.4 주요 API 엔드포인트
```
GET /api/v1/districts/{sggCode}/member    # 선거구 → 의원 조회
GET /api/v1/members/{monaCode}            # 의원 상세
GET /api/v1/members/{monaCode}/bills      # 법안 발의 내역
GET /api/v1/members/{monaCode}/votes      # 표결 내역
GET /api/v1/members/{monaCode}/attendance # 표결 참여율
GET /api/v1/members/search                # 통합 검색
```

## 8. 페이지 구성

### 8.1 메인 페이지 (`/`)
- 전체 화면 지도 (선거구 경계 오버레이)
- 상단 검색 바 (의원 이름·정당·지역)
- 선거구 클릭 시 의원 정보 사이드 패널 슬라이드

### 8.2 의원 상세 페이지 (`/members/{monaCode}`)
- 프로필 헤더 (사진, 이름, 정당, 선거구, 당선 횟수)
- 탭 구성:
  - **활동 요약**: 표결 참여율 게이지, 발의 건수, 핵심 지표
  - **법안**: 발의 법안 목록 (필터: 상태, 연도)
  - **표결**: 표결 내역 (찬성/반대/기권/불참)
  - **재산·전과**: 재산 신고 내역, 전과 기록

### 8.3 검색 결과 페이지 (`/search`)
- 검색 결과 리스트 (의원 카드)
- 필터: 정당, 지역, 당선 횟수

### 8.4 선거구별 페이지 (`/districts/{sggCode}`) - SEO용
- 선거구 정보 + 해당 의원 프로필
- SSG로 빌드 타임에 사전 생성
